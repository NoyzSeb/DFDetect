# Project Structure and Instructions

## 1. Configuration (configs/)
This directory manages hyperparameters using Hydra. It replaces hardcoded constants.

- **config.yaml**: The master configuration file. It composes the sub-configs (model, data, trainer) into a single run configuration.
- **model/**: Contains YAML files for model architectures (e.g., resnet50.yaml, vit.yaml). Defines hidden sizes, layer counts, and activation functions.
- **data/**: Defines dataset paths, batch sizes, and num_workers.
- **trainer/**: Controls the PyTorch Lightning Trainer (e.g., max_epochs, gpus, gradient_clip_val).

## 2. Data Storage (data/)
- **raw/**: Immutable. Stores the original dataset downloaded from the source. You must never modify files here programmatically; treat them as read-only to ensure reproducibility.
- **processed/**: Stores artifacts generated by your code (e.g., tokenized text, resized images, .pt tensors). These should be reproducible from raw data.

## 3. Source Code (src/)
This is the core Python package.

- **models/** (The Architecture): Contains pure PyTorch nn.Module classes. These files should not depend on PyTorch Lightning. This ensures your model architecture is portable and can be used in non-Lightning contexts (e.g., deployment).
- **systems/** (The Task): Contains the LightningModule. This class wraps the architecture from models/. It defines how the model learns (loss calculation, training_step, validation_step, optimizer configuration).
- **datamodules/**: Contains LightningDataModule classes. This encapsulates the full data pipeline: downloading, splitting (train/val/test), and wrapping data in DataLoader instances.
- **utils/**: Helper functions for non-ML logic, such as custom metrics, logging setup, seed setting for reproducibility, or file I/O.

## 4. Logging & artifacts (logs/)
Generated automatically during training.
- Contains subfolders for each experiment run.
- Stores model checkpoints (.ckpt), TensorBoard/WandB logs, and a snapshot of the Hydra configuration used for that specific run.

## 5. Scripts
- **train.py**: The entry point. It instantiates the DataModule and LightningModule from the configuration and calls trainer.fit().
- **eval.py**: Loads a trained checkpoint and runs the test set using trainer.test().

## 6. Support
- **notebooks/**: For prototyping only. Do not put production training logic here.
- **tests/**: Unit tests. Critical for verifying that your data shapes match and your model performs a forward pass without error before starting a long training job.
